В этом посте я расскажу о том как можно самостоятельно стать фронтендером, максимально быстро, какие технологии учить, в каком порядке и на каких ресурсах.

Покажу как можно практиковаться и набивать скилл. Где искать инфу. Как оформлять свое резюме и профили для рекрутеров, как пройти собес и устроиться в крутую компанию.

Все эти шаги я прошел на личном опыте, из них я собрал самое важное для вас. Чтобы вы могли пройти этот путь без проблем.

Примерно наш путь будет складываться таким образом:

1. Тулинг - установка всех необходимых для работы программ
2. Базис - основы веба HTML, CSS
3. Верстка - углубление в верстку и CSS
4. JavaScript / DOM - разобраться с базовым JS и научиться манипулировать страничкой
5. React - изучить фреймворк
6. Устроиться на работу
7. ...продолжать учиться постоянно

Каждый пункт включает в себя кучу материалов и практики - ведь без практики все это не имеет смысла.

И помните, дорогу осилит идущий. Поехали!

# Тулинг

Ниже перечилю программы которыми пользуюсь лично я, вы конечно можете юзать то что нравится

- [VS Code](https://code.visualstudio.com/) - редактор кода
- [Figma](https://www.figma.com/downloads/) - дизайн маеты
- [NodeJS + NPM](https://nodejs.org/en/) - для запуска локального сервера и окружения разработки и пакетный менеджер JS
- [Focus To-Do](https://www.focustodo.cn/) - таск трекер
- [Brave Browser](https://brave.com/) - браузер основанный на chromium который блокирует реклами и ослеживание

На этом все, для начала этого пака хватить для продуктивной работы.

# Понять основы HTML, CSS, JS и устройство интернета

1. Что бы пронять как все работает читаем статью - [Как раотает веб?](https://developer.mozilla.org/ru/docs/Learn/Getting_started_with_the_web/How_the_Web_works) там же есть и дополнительные стати по этой теме

2. Дальше разбираемся с HTML читаем эту статейку - [Основы HTML](https://developer.mozilla.org/ru/docs/Learn/Getting_started_with_the_web/HTML_basics)

3. Читаем сделующую статейку по CSS - [Основы CSS](https://developer.mozilla.org/ru/docs/Learn/Getting_started_with_the_web/CSS_basics)

4. И немного практики. Интерактивный курс [Основы HTML и CSS](https://htmlacademy.ru/courses/297) - проходим его и выполняем все задания

5. Практика. Верстаем эти макеты: [Макет 1](https://www.figma.com/file/OYdqbX9ilhluXys1JdKoC7/HTML-Level-1), [Макет 2](https://www.figma.com/file/hzcfTXSRs1tcj3JEV1yqKW/Lesson-1-Base-HTML-CSS), [Макет 3](https://www.figma.com/file/NBF7w0AxnbMb5DNsczeyFP/WP-Indigo?node-id=0%3A1), [Макет 4](https://www.figma.com/file/OYdqbX9ilhluXys1JdKoC7/HTML-Level-1)

# Верстка

1. Выучить все базовые CSS селекторы и правила. [Селекторы которые вы должны запомнить](https://code.tutsplus.com/ru/tutorials/the-30-css-selectors-you-must-memorize--net-16048), [CSS правила и еще интересное по CSS](https://webdesign-master.ru/blog/html-css/2017-01-10-css-rukovodstvo.html)
2. Выучить [БЕМ](https://ru.bem.info/methodology/) + [Видос](https://www.youtube.com/watch?v=lQPs20b3Ey8)
3. Изучить css Fex box - [игруля](http://flexboxfroggy.com/) + [материал](https://www.youtube.com/watch?v=eVZEwEQg4pg)
4. Изучить css Grid layout - [игруля](https://cssgridgarden.com/), [материал статья](https://css-tricks.com/snippets/css/complete-guide-grid/) + [материал видео](https://www.youtube.com/watch?v=MEOR2b69Pl4)
5. Адаптивная верстка, медиазапросы - [норм статейка](https://kokoc.com/blog/chto-takoe-adaptivnaya-vyorstka-sajta-primery/) + [статейка](https://developer.mozilla.org/ru/docs/Web/CSS/Media_Queries/Using_media_queries) + [видос](https://www.youtube.com/watch?v=M-xc1EOMOIE)
6. Изучить [Bootstrap](https://getbootstrap.com/) - в первую очереть сетку и потом все остальное, [дока на русском](https://bootstrap-4.ru/)
7. Заюзать самый простой сборщик проекта [Parcel](https://parceljs.org/) это даст вам +100 к скорости работы и +100 к скорости изучения. [Инструкция](https://ru.parceljs.org/getting_started.html)
8. Изучить препроцессор [SASS](https://sass-lang.com/) - [основы](https://sass-scss.ru/guide/) этого пока хватит
9. Практирка. [Макет 1](https://www.figma.com/community/file/892358789568947362), [Макет 2](<https://www.figma.com/file/Wcy4LyehlSozF5lZEnQQq8/Cloud-Hosting-Landing-Page-(Community)?node-id=0%3A1>), [Макет 3](https://www.figma.com/file/kyMjaFeYuOXUF0JrU78fs7/Food-POS-Dark-Tablet-Device), [Макет 4](https://www.figma.com/file/jVYaRnSgd6UmpTNfazirHL/Premium-Car-Blog-Template), [Макет 5](https://www.figma.com/file/D7I4z2hwde0Tjp4QaOgxk0/Landing-Page-Concept-For-Data-Processing-Company), [Макет 6](https://www.figma.com/file/OwmXhezyNEHJb4WTSTa5Ao/Figmaland-Business-Landing-page), [Макет 7](https://www.figma.com/file/AJL1BoDlqi6AU17y1JPVzl/MNTN-Landing-Page), [Макет 8](https://www.figma.com/file/Gelkc2cspUVlUp7rrdcJUr/%F0%9F%8E%A8-Creative-Website-Template-for-Figma-Community), [Макет 9](https://www.figma.com/file/FMR1FiRnA62q5T8RzEC1Wn/Woo-Landing), [Макет 10](https://www.figma.com/file/MgBTtD2S2kTgXT73D1nPVq/Flowbase-Real-Estate-Template), [Макет 11](https://www.figma.com/community/file/827587049636330363) - на этом навык верстки думаю будет освноен. И помните что не адаптивная верстка - это не верстка!
